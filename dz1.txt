Заголовки ответов:

Date  -  содержит дату и время, в которое сообщение было создано.

Expires  -  содержит дату/время, по истечении которой ответ сервера считается устаревшим. На ютубчике стоит дата Mon, 01 Jan 1990 00:00:00 GMT, очевидно, какая-то шляпа.

Pragma  -  заголовок, зависящий от реализации, который может иметь различные эффекты в цепочке запрос-ответ. Он используется для обратной совместимости с кешами HTTP / 1.0, где заголовок Cache-Control HTTP / 1.1 ещё не присутствует.

Server  -  описывает программное обеспечение, используемое исходным сервером, обработавшим запрос.

Strict-Transport-Security  -  заставляет бразуер использовать HTTPS вместо HTTP. Надежненько.

Content-Type  -  используется для того, чтобы определить тип контента.

X-Content-Type-Options  -  типы, обяъвленные в Content-Type, должны соблюдаться и не изменяться.

Content-Encoding  -  используется для сжатия медиа-типа.

Content-Length  -  указывает размер отправленного получателю тела объекта в байтах.

Last-Modified  -  содержит дату и время, в которую, по мнению удалённого сервера, запрашиваемый ресурс был изменён.

Заголовки запросов:

Accept  -  указывает, какие типы контента, выраженные как MIME типы, клиент может понять.

Accept-Language  -  сообщает серверу, какие языки клиент понимает (русский, английский там, всё такое)

Host  -  содержит имя домена, для которого предназначен запрос и, опционально, номер порта.

Authorization  -  включает в себя данные пользователя.

Origin  -  показывает откуда будет производиться загрузка, содержит в себе имя сервера.

Referer  -  содержит URL исходной страницы, с которой был осуществлён переход на текущую страницу.

X-YouTube-Client-Version  -  версия клиента ютуба.

X-Goog-Visitor-Id  -  ну, судя по всему, мой ID.

User-Agen  -  позволяет серверам и сетевым узлам идентифицировать приложение, операционную систему запрашивающего пользовательского агента. 

Accept-Encoding  -  объявляет, какую кодировку контента, обычно алгоритм сжатия, может понять клиент.
